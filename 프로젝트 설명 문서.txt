 기존 : react(html, css, javascript)를 이용한 쇼핑몰 사이트 구현과 jsp 와 tomcat, sqlite를 이용해서 간단한 로그인, 회원가입 기능(정규식 정도까지)을 만들었음

 업데이트 : Springboot 와 Sqlite 를 이용하여 로컬 환경에서 로그인과 회원가입, 게시물 등록, 사용자의 정보 수정, 아이디/비밀번호 찾기, 비회원/회원 상태일때 해당하는 주소를 interceptor로 제한하는 기능과 같은 것들을 더 만들었음.

 1 로그인 시 
  (1) 비밀번호 암호화된것 검증 matches()
  (2) CAPSLOCK 표기 => 미구현

 2 회원가입, 회원탈퇴 기능
  (1) 회원가입
	=> 비밀번호 암호화(spring security 이용) encode()
	=> 이메일 정규식 최신화 필요(미구현, 추후예정)
  (2) 회원탈퇴
	=> 알림창 

 2 게시물 CRUD : 파일업로드까지

 3 사용자의 정보 수정 : 회원가입과 비슷하게 구현
   1) 초기 데이터 값을 input value 값에 넣기 위해 useEffect를 사용했음
   2) 기본키인 id 값은 변경 불가하므로 휴대전화번호와 이메일의 중복 검사는 삭제했음

 4 아이디, 비밀번호 찾기 : 이메일 인증 기술을 이용해 구현
  -> 1) 이름 속성 추가 후 이름과 이메일을 통해 인증(아이디) => 웹페이지에서만 보이게 구현
	=> switch 구문 이용했음, useState 를 통해 웹페이지의 HTML 태그에 직접 접근

  -> 2) 아이디와 이메일을 통해 비밀번호 인증(비밀번호)
	=> sessionID가 아닌 sessionIDPwRevise 를 사용. 
	=> 비밀번호 수정을 통해 기존 비밀번호를 save 하여 업데이트 하는 방식으로 동작
	=> 이메일 인증 기술 사용(미구현, 추후예정)

 5 비회원/회원 상태일 때 접속 제한기능
 (1) front-end : react app.js 에서 router, render, navigate to 기능으로 제한
 (2) back-end : interceptor 로 해당하는 주소 제한하는 기능
  - 회원일시 회원가입 페이지 막기, 개인정보 수정 전 비밀번호 인증을 통해 막기
  - 비회원일시 마이페이지, 개인정보수정 페이지 막기
  - 각각의 react jsx 에서 useeffect 를 사용 => headers 값의 유무를 검사하여 로그인 유무값을 판단 or ?
